# リポジトリ横断ルール

エージェントと人間がこのリポジトリを変更するときの正本。プロジェクトの目的・起動・検証コマンドは [README.md](./README.md) を読む。

## 日本語文書・コメント

TW3級（テクニカルライティング試験3級）水準で書く。

- 一文一義。1つの文には1つの主旨だけを書く
- 能動態を優先する
- 用語を統一する。同じ対象を別の語で言い換えない
- 曖昧語を排除する。「適宜」「など」を判断基準の代わりに使わない
- 読み手基準で書く。読み手がそのまま実行・判断できる形にする

## 品質保証

- 成果物の品質保証は `/verification-loop` スキル（実装×独立レビューの敵対的ループ）で行う
- 品質ゲートは production (0.93) を既定とする
