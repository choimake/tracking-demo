# 製品contract

この文書はtracking-demoが提供する機能と非対応範囲の正本です。
E2Eによる検証状態は[E2E Coverage Matrix](./docs/e2e-coverage-matrix.md)で管理します。

## 対象

tracking-demoは、計測タグ管理の実現性を検証します。
管理画面はタグ設置、CVイベント、ラベル、レポートを扱います。
顧客LP役のデモサイトは、計測タグを直接設置したMPAとSPAを再現します。

`ws`はworkspace IDです。
workspace IDはHitを計測設定へ紐付けます。
workspace IDはユーザーIDではありません。

## タグ設置

- 管理画面はworkspace IDを含むタグスニペットを発行します。
- スニペットは`tdDataLayer`を使い、`tracker.js`を非同期で読み込みます。
- `tracker.js`の取得中もHTML解析を継続します。
- 利用者はスニペットをページの`<head>`へ設置します。
- 管理画面はスニペットをクリップボードへコピーします。
- 管理画面はGTMのカスタムHTMLタグへ設置する手順を案内します。
- テスト発火は実際の計測受信経路を使ってタグの動作を確認します。

このデモは`tracker.js`を計測サーバーから配信します。
CDN配信は本番化で扱います。

## CVイベント

管理画面はCVイベントの作成、参照、更新、削除に対応します。
各イベントは名前と任意の説明を持ちます。
利用者はイベントを有効化または無効化できます。
利用者はイベント単位でテスト発火を実行できます。

次のトリガーを扱います。

| トリガー       | 入力例                | 発火条件                                             |
| -------------- | --------------------- | ---------------------------------------------------- |
| URL到達        | `url:/order/complete` | パスが指定値に一致する                               |
| クリック       | `click:.add-to-cart`  | 指定したCSSセレクターの要素をクリックする            |
| ページ滞在時間 | `time_on_page:60`     | 読み込み後に指定秒数が経過する                       |
| スクロール率   | `scroll:50`           | 縦方向のスクロールが指定率へ到達する                 |
| 離脱インテント | `exit_intent`         | デスクトップでポインターがビューポート上端から外れる |

無効なイベントは配信設定から除外します。
計測サーバーも無効なイベントのHitを破棄します。
管理画面は無効なイベントの件数を0件と表示します。

管理画面はイベントごとの直近7日間の発火件数を一覧表示します。
一覧は名前、説明、トリガー、件数、状態、操作を表示します。
利用者はイベント名またはトリガーで一覧を絞り込めます。
一覧は検索欄への入力と同時に更新します。

## 計測タグの動作

- 計測タグはpageviewとeventを送信します。
- MPAの読み込みとHistory APIによるSPAのパス変更をpageviewとして扱います。
- `tdDataLayer.push({ event: "tracker.pageview" })`による手動pageviewを扱います。
- 計測タグは、読み込み前に`tdDataLayer`へ追加された項目を初期化時に再生します。
- キュー再生済みのpageviewは重ねて送信しません。
- 同じページへタグを二重に設置した場合は、2つ目以降の読み込みを無視します。
- History APIの自動検知と手動pageviewを併用できます。
- 併用時は、同じ遷移の重複送信を抑止します。

デモサイトと計測サーバーは別オリジンです。
計測タグはCORSを使って設定を取得します。
計測タグは`sendBeacon`を優先してHitを送信します。

## 匿名識別と集計

| Cookie    | Hitの項目 | 意味                   | 有効期間                |
| --------- | --------- | ---------------------- | ----------------------- |
| `_td_vid` | `vid`     | 匿名の再訪識別子       | 1年。Hitごとに延長する  |
| `_td_sid` | `sid`     | 匿名のセッション識別子 | 30分。Hitごとに延長する |

計測タグは顧客サイトの文脈でCookieを書き込みます。
Cookieには`Path=/`と`SameSite=Lax`を指定します。
HTTPデモでは`Domain`と`Secure`を指定しません。
JavaScriptで読み書きするため`HttpOnly`を指定しません。

`vid`と`sid`は認証、認可、アクセス制御に使いません。
Cookieを書き込めない場合も計測タグはHitを送信できます。
この場合は再訪識別とセッション継続が成立しません。

テスト発火は`test: true`のHitとして記録します。
テスト発火では`vid`と`sid`を空文字にします。
テスト発火は計測件数とレポートから除外します。

直近7日間の件数は、計測サーバーのローカル日付を基準にした7暦日で集計します。
イベント一覧とレポートは同じ集計期間を使います。

## ラベル、レポート、レコメンド

- 管理画面はラベルの作成と削除に対応します。
- 管理画面はイベントへラベルを設定します。
- レポートは計測結果をイベント単位で表示します。
- 管理画面はルールに基づき、無効なイベントの有効化を推奨します。
- 設定画面はモーダルで表示します。
- 設定画面はworkspace ID、workspace名、計測エンドポイントを表示します。
- 利用者は設定画面でworkspace名を更新できます。

## 非対応範囲

次の計測機能は対象外です。

- 実際のGTMコンテナとの接続
- CMPによる同意管理
- 3rd-party Cookie、フィンガープリント、クロスドメイン計測
- UTM、日付変更時のセッション分割、`session_start`、`session_number`
- ログインユーザーと紐付ける`user_id`
- ページへのラベル設定
- ハッシュルーティングによるページ遷移の検知
- クエリだけが変わるURLの再評価
- モバイル端末での離脱インテント発火

ブラウザ別の検証済みcontractと未検証範囲は[E2E Coverage Matrix](./docs/e2e-coverage-matrix.md)で管理します。

## 本番化の範囲外

- 管理画面と管理APIに認証・認可を実装しません。
- 計測エンドポイントにレート制限とデータ保持期限を実装しません。
- 計測APIのCORSは許可元をworkspace単位で制限しません。
- HTTPS、CSP、セキュリティヘッダーを実装しません。
- ファイルベースの疑似DBをRDBへ置き換えません。
- Safari ITPを含むブラウザ方針によるCookie有効期間の短縮を制御しません。
- 商用環境の可用性、性能、監視、集計基盤を保証しません。
